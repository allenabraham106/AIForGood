# Technical Architecture — For the Pitch (Person 5)

Short, judge-facing explanation of how the system works. Use these points when presenting.

---

## High-level flow

1. **User picks a scenario** (e.g. “Greeting a New Resident”) from 4 PSW-focused lessons.
2. **Play lesson** triggers the audio pipeline: dialogue → cultural tip → key phrase (repeated) → reflection question. All spoken in sequence by the browser.
3. **Reflection question** is either **generated by Gemini** (when the API is configured) or **read from the scenario** (fallback). The question is short, beginner English, and encouraging.
4. **User taps “Got it”** to mark the lesson complete; that hook will connect to the main app’s progress/tree (Person 2).

---

## Tech stack (what we built with)

| Layer | Technology | Why |
|-------|------------|-----|
| **Frontend** | React 18, Vite | Main app (Branch) and fast dev/build. |
| **Audio** | Web Speech API (browser TTS) | No external TTS API; works offline; we control pacing and sequence. |
| **Reflection questions** | **Google Gemini** (sponsor stack) | One short question per scenario; constrained prompt (max 10 words, beginner English). |
| **Backend** | Vercel serverless | Single serverless function: `POST /api/reflection-question` calls Gemini with scenario context. |
| **Content** | `scenarios.js` | Scenario scripts (dialogue, cultural narration, key phrase, fallback reflection question) in one place. |

---

## Architecture in one sentence

**“The learner hears a scripted dialogue and cultural tip, practices a key phrase, then gets a short reflection question—either from our scenario file or generated by Gemini—all spoken by the browser, with the API running on Vercel.”**

---

## What we deliberately didn’t use (ethics / cultural relevance)

- **No speech-to-text (e.g. Whisper) for this demo** — We’re not recording or transcribing the learner; we’re focused on listening and reflection. (If you have an ethics slide: “We chose not to use [X] because…” — judges remember this.)
- **Browser TTS only** — Keeps the demo simple, private, and runnable without extra APIs; real deployment could add a preferred TTS later.

---

## Handoff points for the rest of the team

- **Person 1:** Use `pipeline.playScenario(scenario, callbacks)` and `onStateChange` for UI (e.g. waveform, active step).
- **Person 2:** Replace the demo’s “Got it” handler with `markComplete(scenarioId)` and wire to the tree/progress.
- **3A (reflection):** Four questions live in `scenarios.js`; Gemini API can override them when configured; prompt context from `buildReflectionPromptContext(scenario)`.

---

## If judges ask “Where’s the AI?”

**“Gemini generates the reflection question. We send the scenario—dialogue, context, key phrase—to our serverless API; Gemini returns one short, beginner-friendly question. If the API isn’t available, we use the four questions we wrote in the scenario file.”**
